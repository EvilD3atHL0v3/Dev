# Standard Operating Procedure (SOP)
## IOC Log Collection Using Custom Executable Tool (Windows Only)

---

### 1. Purpose
This document outlines the standard procedure for using a custom-developed executable (`.exe`) tool to automate the collection of system logs and artifacts for Indicators of Compromise (IOC) analysis during security investigations.

### 2. Scope
This SOP applies to all authorized personnel involved in security monitoring, incident response, and forensic investigations within the organization.

**Platform Limitation:** This procedure is applicable to Microsoft Windows operating systems only (e.g., Windows 10, Windows 11, Windows Server editions). The tool and commands referenced in this document are designed specifically for Windows environments and are not intended for use on Linux, macOS, or other operating systems.

* **Tool Name:** [Insert Tool Name]
* **Version:** [Insert Version]
* **Developer:** Internal (Custom Developed Tool)
* **Source:** Internal Security Team
* **File Hash (SHA256):** [Insert Hash]
* **Description:** This tool is internally developed to automate basic log collection tasks using command-line operations.

### 4. Features and Capabilities
The tool automates the collection of the following:
* Command Prompt history (CMD history)
* Browser history (e.g., Chrome, Edge)
* Basic system information
* Other lightweight forensic artifacts as required

### 5. Prerequisites
* Administrative privileges on the target machine
* Access to the approved internal tool repository
* Approval from Security Team or relevant authority

### 6. Execution Procedure
1.  Copy the approved executable file to the target machine.
2.  Verify the file integrity by validating the SHA256 hash.
3.  Open **Command Prompt** with administrative privileges.
4.  Navigate to the directory containing the executable.
5.  Execute the tool.
    * **Example Command:** `C:\\Tools\\CustomLogCollector.exe`
6.  Allow the tool to run and complete the automated collection process.
7.  Confirm that output files are generated successfully.

### 7. Data Collected
The tool collects the following data:
* Command Prompt history
* Browser history
* Basic system and user activity logs

### 9. Security and Integrity Controls
* Tool is internally developed and maintained by the Security Team.
* Validate file hash before execution.
* Restrict usage to authorized personnel only.
* Store source code securely (if applicable).
* Document any antivirus or endpoint detection exceptions (if required).

### 10. Risks and Considerations
* Execution may trigger security alerts due to data collection behavior.
* Requires elevated privileges.
* May access sensitive user data (e.g., browser history).
* Ensure proper authorization before execution on user machines.

### 11. Whitelisting Consideration
This custom-developed tool is approved for IOC log collection. Repetitive alerts generated by its execution may be considered for whitelisting, subject to validation, security review, and formal approval.
